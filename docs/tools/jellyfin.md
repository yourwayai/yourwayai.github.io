# Jellyfin: 夺回影音主导权的自由多媒体串流中枢

## ✨ 核心特性

### 1. 永不妥协的纯粹开源体验 (No Premium, Zero Strings)
区别于同生态赛道下老牌的商业竞争对手（诸如同样有名的 Emby 与 Plex）为了盈利而设定那些令人厌烦的“锁定特色功能并按月售卖许可密钥 (Plex Pass)”等分层收费机制。Jellyfin 打从诞生初始就剔除了一切内置消费墙和隐藏高级解锁包的念头，例如**核心硬件视频转码等高端商规特性，一出厂就默认对所有用户社群毫无保留地免费完整开启**。

### 2. 恐怖的极至跨终端覆盖率 (Clients for whatever you own)
这是真正意义上打造了一套全终端实时串流播控大局域网生态体系。不仅仅局限在基于网页浏览器端的高清解码呈现，它的信号触角更是跨越平台鸿沟，下探覆盖到了 Android、iOS 主流操作系统的手机及平板阵营；并直接延伸到了家庭影音生活大屏幕矩阵领域，诸如主流生态 Android TV、Apple TV 或是廉价盒子如 Amazon Fire TV 与 Roku 终端。

### 3. 暴力的前沿硬件级转码池化体系 (Hardware Transcoding Engine)
当你带着移动便携设备在外出差穿梭于复杂网络下，企图去点播一台位于家中百 T 硬盘里重达 60 多 GB 码率甚至采用 4K HEVC 制式的顶级蓝光未压缩电影时，Jellyfin 强悍的后台会瞬间激发硬件智能自适应转码降频压缩引擎。系统天生对于时下诸如 Intel 系列 (QSV/Quicksync核心模块), Nvidia 绿厂卡列 (NVENC加速) 与 AMD 平台 (AMF/VAAPI渲染) 这三大芯片商的压榨指令层代码全都进行了极致化微操磨合调优封装。做到实时抽取、剥离轨、高度缩放压流并推给弱网客户端，在保持不顿卡的动态串流画面时依旧拥有可被接纳的分发画质。

### 4. 高阶的元数据全自动化精准刮削 (Auto Meta scraping & Sync)
即使你的原始下载目录里只是一群由不规范数字英文字母堆叠构成的野生乱码视频文件名。没关系！借用这套体系内部极为庞大恐怖的内置式官方搜图海报插件组系统库以及代理钩子抓取引擎。Jellyfin 能够完全自动执行后台匹配洗稿，实时去如 IMDB、TMDB 等国际公立大型影视剧作品总数据库关联站点群内下载最完美的本地替换版精修系列海报封套照、高清大字头图 Logo、主干详细剧情叙事梗概和甚至能同步精确到每个演员头像简历。

## 🚀 极速部署方案

要搭建起这么一套无与伦比的“家用媒体重器”，在现代 Docker 全能容器时代而言非常轻松易上手。它的架构思路十分明朗清晰——仅需拉偏起一个守护指令主容器服务进程，随之正确且安全地授予它用于访问你位于本地亦或者通过网络挂接下来的庞大海量影视仓库的只读 (Read Only) 或增补修改存取操控权限映射通道即可。

```yaml
# 以下提供了一份十分标准的 docker-compose.yml 部署组合清单代码结构
version: '3.5'
services:
  jellyfin:
    image: jellyfin/jellyfin:latest
    container_name: jellyfin
    # 注意安全提示：如果系统宿主体内的原始电影目录存有复杂的安全特权分配链条归属划分，在此务切对应好读写运行所属组的识别数字如 UID/GID (1000:1000)
    user: 1000:1000 
    network_mode: 'host'
    volumes:
      # 规划好底下这三个核心的绑定点作为最底层磁盘数据的路径落脚处，才能坚若磐石地保证往后纵使主容器被手抖摧毁乃至强行升级结构化数据底包重构迭代更新后都不至于会导致先前的观影海报及播放时长位置标签纪录凭空受损：
      - /path/to/your/config:/config #存放内部运作配置信息核心设定
      - /path/to/your/cache:/cache #流媒体转压切片产生时的即时内存缓存碎片收集篓
      - /path/to/your/media:/media #最重要的一项，此处为指向真实藏匿你珍贵影视原片仓库库房位置的光标节点映射锚。
    restart: 'unless-stopped'
```
