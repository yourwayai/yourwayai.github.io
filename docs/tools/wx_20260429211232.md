### 🔪 令人抓狂的“在线格式转换”

在日常学习和工作中，我们总会遇到各种奇葩的文件格式转换需求。比如把网页保存的 WebP 转成 PNG，把冷门的 MKV 视频压成 MP4，或者把一堆 Markdown 文档打包转成 PDF。

当你满怀希望地打开搜索引擎，随便点进一个“免费在线转换”网站时，噩梦就开始了：  
要么提示你“文件大小不能超过 50MB”，要么转换完下载时强迫你关注公众号，甚至还需要充值 VIP 才能去除烦人的水印。

更致命的是**数据隐私问题**。如果你要转换的是公司的机密财务报表、未公开的产品原型，或是个人的私密照片，把这些文件上传到不知名的小网站后台，无异于在互联网上“裸奔”。我们需要一个无限制、不流氓、且绝对安全可控的专属转换工具。

### 🌟 掌控一切的万能转换器 ConvertX

为了打破这种被流氓网站裹挟的局面，开源社区诞生了一款极其硬核的替代方案：**ConvertX**。

这是一款支持自托管的在线文件转换应用，由极客开发者采用最新一代的高性能技术栈（TypeScript + Bun + Elysia）打造。它最大的亮点在于，将各类垂直领域的顶级转换引擎全部打包在了一起。

通过将 ConvertX 部署在你自己的电脑、云服务器或 NAS 上，你就拥有了一个完全属于自己、速度只受你硬盘限制、永远免费且永不泄露数据的“全能格式转换站”。

### 🚀 为什么它能被称为全能王？

ConvertX 绝不是简单套个壳子，它的底层极其扎实，以下是让它脱颖而出的几大核心优势：

* • **集大成者：支持超过 1000 种格式**  
  普通的转换工具往往只专精一项，但 ConvertX 是个彻头彻尾的“缝合怪”（褒义）。它底层集成了几十种业界鼎鼎大名的开源引擎。例如用 FFmpeg 处理几乎所有的音视频格式，用 Pandoc 和 LibreOffice 搞定各种复杂的办公文档，用 ImageMagick 处理各类图片，甚至还内置了 Calibre 引擎专门用来转换电子书（EPUB、MOBI 等）。
* • **极致的批量处理与并发性能**  
  很多在线工具一次只能排队转换一个文件，极其浪费时间。而 ConvertX 原生支持多文件同时上传和批量处理。得益于其底层所使用的 Bun 运行时（一个比 Node.js 更快的现代 JavaScript 运行时），它的 API 响应极快，能充分榨干你服务器的 CPU 性能，瞬间完成大批量的转换任务。
* • **严格的权限控制与多用户支持**  
  既然是部署在服务器上，如何防止别人蹭你的算力？ConvertX 提供了完善的多账号系统和密码保护功能。你可以为团队成员或家人分配独立账号，每个人都有自己的转换历史和工作区。同时，你还可以通过环境变量配置 JWT（JSON Web Token）密钥，确保数据接口的绝对安全。
* • **无忧的自动化存储管理**  
  转换产生的大量临时文件如果不清理，很容易撑爆服务器硬盘。ConvertX 内置了智能清理机制，你可以自定义策略（例如 `AUTO_DELETE_EVERY_N_HOURS=24`），让系统自动巡检并删除超过 24 小时的历史文件，彻底解放运维双手。

### 🛠️ 五分钟部署你的私人转换站

ConvertX 提供了非常完善的 Docker 镜像，无论你是 Windows/Mac 本地用户，还是 Linux 服务器极客，都可以通过 Docker Compose 轻松一键拉起服务。

以下是具体的实战部署代码，请将它们保存为 `docker-compose.yml` 文件：

```
# docker-compose.yml 配置文件  
services:  
  convertx:  
    image: ghcr.io/c4illin/convertx:latest  
    container_name: convertx  
    restart: unless-stopped  
    ports:  
      - "3000:3000"  # 映射本地 3000 端口  
    environment:  
      # ⚠️ 强烈建议修改为你自己的长随机字符串  
      - JWT_SECRET=YOUR_SUPER_SECRET_KEY_HERE_1234  
      # 如果你在本地无 HTTPS 环境下访问（如局域网 IP），必须取消下面这行的注释  
      - HTTP_ALLOWED=true  
      # 自动清理 24 小时前的文件  
      - AUTO_DELETE_EVERY_N_HOURS=24  
      # 限制并发进程数，0 为无限制，建议根据服务器 CPU 核心数设置  
      - MAX_CONVERT_PROCESS=4  
    volumes:  
      # 将产生的数据持久化挂载到当前目录的 data 文件夹下  
      - ./data:/app/data
```

**启动说明与避坑指南：**

1. 1. 在 `docker-compose.yml` 所在的目录下，打开终端运行以下命令启动服务：

   ```
   docker compose up -d
   ```
2. 2. 服务启动后，打开浏览器访问 `http://localhost:3000`（或你的服务器IP:3000）。
3. 3. **关键提醒**：第一次打开时，请务必**立即注册第一个账号**。因为在没有初始数据时，任何人都可以注册成为管理员。
4. 4. **权限报错处理**：如果你在启动时遇到 `unable to open database file` 的错误提示，通常是因为 Docker 挂载出来的 `./data` 目录没有读写权限。你只需要在宿主机终端执行 `chown -R $USER:$USER ./data` 即可完美解决。
